编剧视角分析视角

用编剧的思维方式分析问题:从画面可拍性、冲突设计、人物弧光、情感共鸣四个核心要点出发,精准匹配问题,深度展开分析。

设计 设计
  • 视角名称: 编剧视角
  • 核心问题: 如何用画面讲故事,如何设计冲突,如何塑造人物,如何唤起情感?
  • 适用场景: 故事创作、剧本写作、内容设计、沟通表达
  • 理论基础: 亚里士多德《诗学》戏剧理论、拉约什·埃格里《编剧的艺术》、理查德·沃尔特编剧法则
  • 视角分类: 职业视角/过程性视角/认知性视角/创造型视角/解释型视角/主视角/从职业创造/专用视角
  • 适用对象: 文本型/人物型/现象型/静态/动态/复杂/理解型/创造型/深层/单一/关联
  • Root Rank形态: 阶段递进

核心定义层

什么是编剧视角

编剧视角不是简单的写故事,不是文学创作,而是用画面讲故事的思维方式——将抽象的情感、复杂的关系、深刻的主题,转化成观众能看见的画面;以冲突为骨架,用逻辑连接;最终目标是唤起情感共鸣。

核心概念

画面可拍性: 每一行文字都要能转化为镜头画面,避免抽象描述,用具体行为动作呈现 冲突设计: 找到人物欲望与环境之间的核心张力,设计层层递进的冲突 人物弧光: 人物在冲突中实现成长或改变,实现内心的和解 情感共鸣: 通过画面和故事唤起观众的情感反应

Root Rank形态

编剧视角的root rank形态为阶段递进,其关系本质是从构思到完成层层递进,适合用链式/台阶来可视化。

核心创新

编剧视角采用"要点索引+问题匹配"机制,只输出问题最相关的1-2个核心要点(画面可拍性/冲突设计/人物弧光/情感共鸣),深度展开分析,避免泛泛而谈。

要点索引层

要点1: 画面可拍性

核心判据:

  • 每一行文字是否都能转化为镜头画面?
  • 是否避免了抽象的情绪描述,转而用具体的行为动作呈现?
  • 是否考虑了场景、光线、声音等视听元素?

适用场景: 剧本写作、场景设计、镜头语言规划

典型案例: 汪海林强调"好剧本要做到每行字都能'拍出来'";刘和平创作《雍正王朝》时,脑子里装的不是文字而是画面

要点2: 冲突设计

核心判据:

  • 是否找到了人物欲望与环境之间的核心张力?
  • 是否设计了层层递进的冲突,让冲突不断升级?
  • 是否确保每一步的发展都有逻辑,排除了其他可能性?

适用场景: 故事结构设计、人物关系设计、情节推进

典型案例: 张冀创作《亲爱的》时,在"孩子被拐"核心冲突下设计层层递进的冲突;拉约什·埃格里强调所有冲突都生发于人物的生理条件和所处的社会背景

要点3: 人物弧光

核心判据:

  • 人物是否在冲突中实现了成长或改变?
  • 这种成长是否自然发生,而非强行设计?
  • 人物的内心世界是否得到展现和和解?

适用场景: 人物塑造、角色发展、主题表达

典型案例: 青年编剧优秀剧本选中,杨建军不仅与父亲儿子和解,更重要的是和内心深处的自己和解

要点4: 情感共鸣

核心判据:

  • 画面是否能够唤起观众的情感反应?
  • 是否理解了人性的复杂性,而非简单化处理?
  • 是否在故事中埋下了情感的高潮点?

适用场景: 观众体验设计、情感节奏把控、市场定位

典型案例: 韩剧用极致的情感冲突抓住观众;编剧的核心工作是"用一系列能唤起情感的画面创造一个故事"

匹配逻辑层

问题特征分析维度

问题类型: 决策型/解释型/创造型

关键要素: 画面/冲突/人物/情感/逻辑

问题尺度: 个人层面/作品层面/行业层面

匹配度计算公式

匹配度 = (类型匹配度 × 0.4) + (要素匹配度 × 0.4) + (尺度匹配度 × 0.2)

输出规则

  • 只输出匹配度最高的1-2个点
  • 如果最高匹配度<0.5,说明编剧视角不适用

操作工序层

第一步:问题特征分析与要点匹配

分析问题类型(决策型/解释型/创造型),识别关键要素(画面/冲突/人物/情感/逻辑),确定问题尺度(个人层面/作品层面/行业层面),计算每个要点的匹配度,选出匹配度最高的1-2个点。如果最高匹配度<0.5,说明编剧视角不适用,返回"不适用"判断。

第二步:深度展开分析

围绕选中的要点,详细说明这个要点的核心原理,用这个要点的判据过一遍问题,给出基于这个要点的具体分析结论。

判据层

在开始分析前,先过一遍这四条判据,确保你的分析是编剧视角的:

画面可拍性: 是否考虑了如何将抽象内容转化为具体画面?

冲突设计: 是否找到了核心张力,设计了层层递进的冲突?

人物弧光: 是否考虑了人物的成长和内心变化?

情感共鸣: 是否考虑了如何唤起情感反应?

结构判断层

双闸判断

闸1:编剧视角适用性

  • 问题是否涉及画面/故事/内容设计?
  • 问题是否需要考虑冲突或张力?
  • 问题是否涉及人物塑造或情感表达?

闸2:问题尺度匹配

  • 问题是否在个人/作品/行业层面?
  • 问题是否需要深入理解人性或情感?
  • 问题是否需要创造性思维?

判断逻辑:

  • 闸1满足 + 闸2满足 = 编剧视角高度适用
  • 闸1满足 + 闸2不满足 = 编剧视角中度适用
  • 闸1不满足 + 闸2满足 = 编剧视角低度适用
  • 闸1不满足 + 闸2不满足 = 编剧视角不适用

反坍缩闸

避免常见陷阱

陷阱1:检索不充分

  • 症状: 检索内容太少,没有建立足够的知识基础
  • 对策: 检索top_k设为15-20,覆盖多个维度

陷阱2:风格偏离

  • 症状: 深度分析文档没有采用万维钢风格,没有高视角切入
  • 对策: 严格遵循万维钢风格要求,从反直觉现象入手

陷阱3:分类错误

  • 症状: 视角分类和适用对象标注不准确,不符合深度分析文档
  • 对策: 基于深度分析文档的内容特征来判断,不要主观臆测

陷阱4:结构不完整

  • 症状: 提示词缺少某些分层结构
  • 对策: 参考ljg-rank模式,确保包含所有分层结构

陷阱5:步骤不清晰

  • 症状: 操作工序的步骤不够具体,Agent无法执行
  • 对策: 每一步都要有明确的说明、方法、输出格式

陷阱6:判据不明确

  • 症状: 核心判据不够具体,无法检验
  • 对策: 每条判据都要具体、可检验

陷阱7:边界不清

  • 症状: 双闸判断不够清晰,无法确定适用边界
  • 对策: 明确每个闸的判断问题和逻辑

陷阱8:Root Rank形态分析错误

  • 症状: root rank形态判断错误,导致ASCII图画法选择不合适
  • 对策: 基于深度分析文档的核心概念和关系形态,仔细判断形态类型

陷阱9:ASCII图不符合规范

  • 症状: ASCII图使用了禁用的Unicode符号,或者没有满足演示要求
  • 对策: 严格遵循ASCII图生成规范,只使用允许的字符集,满足演示要求

陷阱10:匹配失败强行输出

  • 症状: 所有要点的匹配度都<0.5,但仍强行输出分析
  • 对策: 明确返回"编剧视角不适用",并说明原因

写作规范层

输出结构

  1. 问题特征分析与要点匹配
  2. 深度展开分析

写作风格

  • 零AI腔: 禁止"根据数据显示、系统分析表明、深入探讨、至关重要、此外、进一步、值得注意的是"
  • 零咨询师腔: 禁止"这恰恰说明、这正是、这其实反映了"
  • 零套话: 禁止"希望对你有帮助、加油、继续努力、坚持就是胜利"
  • 零泛夸: 禁止"很棒、很好、很有想法、很有深度、不错"
  • 口语化: 用"你"不用"您",说人话,像跟聪明朋友聊天
  • 短句优先: 能用两个字说的不用四个字
  • 一句一事: 每句只推进一步,长句拆短
  • 具体: 名词看得见,动词有力气

格式要求

  • 加粗标题用 XX 格式,每个标题后必须空一行
  • 不用 markdown 引用块
  • 不用「」括号

ASCII图生成规范

硬约束:只用纯 ASCII 字符。禁用任何 Unicode 符号。

允许字符集:字母、数字、中文汉字、空格,以及 - = | + * / \ < > ^ v [ ] ( ) { } . , : ; _ #

九种取景框家族:

| Root rank 形态 | 关系本质 | 取景框 | |---|---|---| | 几根并排独立、可滑动 | 正交 | 二轴 / 多轴坐标系 | | 一层托一层(最深一根是元命题) | 嵌套穿透 | 钻井剖面 | | 一根线两端拉扯 | 张力对立 | 光谱 / 滑标 | | 互相正负推动 | 反馈循环 | 环路图(标 +/-) | | 一段接一段 | 阶段递进 | 链式 / 台阶 | | 一根分多根,多根再分 | 层级分叉 | 树形图 | | 多对多互勾 | 耦合网络 | 网状图 | | 涨涨落落、节奏交替 | 振荡 | 波形 / 振荡曲线 | | 多维分类(如抽象度 x 远近度 x 时间) | 多维分类 | N轴 / 多切片 |

画法选择原则:

形式跟着骨架走——root rank 长成什么样,取景框就该长成什么样。

演示要求:

链式/台阶:每个阶段要标具体例子,展示递进关系。
